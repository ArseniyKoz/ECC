# Криптография на эллиптических кривых (Elliptic Curve Cryptography)

## Часть 1. Эллиптические кривые над полем вещественных чисел

### Эллиптические кривые

В общем случае эллиптические кривые представляют собой множество точек, удовлетворяющих уравнению:

$$y^{2} + a_{1}xy + a_{2}y = x^{3} + a_{3}x^{2} + a_{4}x + a_{5} \quad\quad (1)$$

Однако нас будет интересовать **частный вид** этого уравнения:

$$\left\{ \begin{array}{r}
y^{2} = x^{3} + ax + b \\
4a^{3} + 27b^{2} \neq 0
\end{array} \right.$$

Второе условие необходимо, чтобы избежать вырожденных (особых) кривых.

### Примеры эллиптических кривых

#### Кривая secp256k1 (Bitcoin и Ethereum)

Пример кривой, которая используется в криптовалютах Bitcoin и Ethereum:

$$y^{2} = x^{3} + 7 \quad\quad \text{(secp256k1)}$$

### Построение группы на эллиптических кривых

Для создания эффективной модели шифрования необходимо построить **группу для эллиптических кривых**.

Основные элементы группы:
- **Элементы группы**: точки на эллиптической кривой
- **Нейтральный элемент**: бесконечно удаленная точка `O`
- **Обратный элемент**: для точки P обратной является точка, симметричная относительно оси Ox
- **Операция сложения**: сумма трех коллинеарных точек P, Q и R равна нулю: `P + Q + R = O`

В такой группе операция сложения является **ассоциативной и коммутативной**, следовательно, наша группа является **абелевой группой**.

### Арифметика на кривых

Исходя из свойств абелевой группы:

$$P + Q + R = O \Rightarrow P + Q = -R$$

При приближении точки Q к точке P получаем:

$$P + P = -R$$

Для кратного сложения точки самой с собой:

$$kP = P + P + \ldots + P \quad \text{(k раз)}$$

где `k` — количество сложений точки P самой с собой для получения точки Q.

### Переход к конечным полям

Проблема непрерывности области определения:

$$\left\{ \begin{array}{r}
X_{\max} = \infty \\
Y_{\max} = \infty
\end{array} \right.$$

требует бесконечной области памяти для вычислений с плавающей точкой. Для решения этой проблемы переходим к **конечным полям** и рассматриваем только целые точки.

Пусть `p` — простое число, а ℤₚ — порождённое им конечное поле. Уравнение кривой принимает вид:

$$\left\{ \begin{array}{r}
y^{2} = x^{3} + ax + b \pmod{p} \\
4a^{3} + 27b^{2} \neq 0 \pmod{p}
\end{array} \right.$$

## Криптография на эллиптических кривых

### Основные принципы

**ECC является асимметричным шифрованием** с следующими характеристиками:

- **Публичный ключ**: точка P на кривой
- **Приватный ключ**: число k
- **Генераторная точка G**: базовая точка с известными координатами

Генераторная точка G выбирается так, чтобы из неё можно было попасть в наибольшее количество точек кривой в данном поле.

### Связь между ключами

Связь между приватным и публичным ключами:

$$P = kG$$

где:
- `k` — **приватный ключ** (количество "прыжков")
- `P` — **публичный ключ** (конечная точка после k прыжков из точки G)

### Задача дискретного логарифмирования (ECDLP)

$$P = kG \quad \Rightarrow \quad k = \frac{P}{G}$$

**ECDLP является вычислительно неразрешимой задачей**:
- Вычислить P, зная k и G — **лёгкая задача**
- Вычислить k, зная P и G — **очень сложная задача**

Не существует субэкспоненциального алгоритма дискретного логарифмирования, что обеспечивает надёжность криптографической системы.

### Примеры задач ECDLP

#### Пример 1
Дано:
- Кривая: $y^{2} = x^{3} - 7x + 10$ в поле $\mathbb{F}_{11}$
- $G = (9, 4)$
- $nG = (8, 2)$

Найти: `n = ?`

#### Пример 2
Дано:
- Поле: $\mathbb{F}_{9679}$
- $G = (6517, 122)$
- $nG = (6686, 3871)$

Найти: `n = ?`

### Реальные криптографические параметры

Для реальных криптографических задач используются очень большие значения `p`. 

**Пример для Ethereum:**

**Координаты точки G:**
```
02 79BE667E F9DCBBAC 55A06295 CE870B07 029BFCDB 2DCE28D9 59F2815B 16F81798
```

**Порядок группы:**
```
FFFFFFFF FFFFFFFF FFFFFFFF FFFFFFFE BAAEDCE6 AF48A03B BFD25E8C D0364141
```

С такими параметрами задача дискретного логарифмирования практически неразрешима.

## Алгоритм Диффи-Хеллмана на эллиптических кривых

### Назначение

Алгоритм используется для **согласования ключей** и обеспечения конфиденциальности передачи данных. Решает проблему: как двум сторонам (Алисе и Бобу) безопасно обмениваться информацией по незащищённому каналу.

### Алгоритм работы

**Подготовка:**
- Определяется конечное поле на эллиптической кривой
- Выбирается генераторная точка G

**Шаги алгоритма:**

1. **Генерация ключей:**
   - Алиса генерирует случайный приватный ключ $d_A$ и вычисляет публичный ключ $H_A = d_A G$
   - Боб генерирует случайный приватный ключ $d_B$ и вычисляет публичный ключ $H_B = d_B G$

2. **Обмен публичными ключами:**
   - Алиса и Боб обмениваются публичными ключами $H_A$ и $H_B$ по незащищённому каналу
   - Перехватчик может получить эти данные, но не сможет определить приватные ключи из-за сложности ECDLP

3. **Вычисление общего секрета:**
   - Алиса вычисляет: $S = d_A H_B$
   - Боб вычисляет: $S = d_B H_A$
   - Значение S будет одинаковым для обеих сторон

**Результат:** Получив общий секретный ключ S, стороны могут обмениваться данными с использованием симметричного шифрования.

### Заключение

Криптография на эллиптических кривых обеспечивает высокий уровень безопасности при относительно небольших размерах ключей, что делает её идеальной для современных криптографических применений, включая блокчейн-технологии и защищённые коммуникации.
